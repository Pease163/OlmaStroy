## User Flows

### 1. Посетитель просматривает проекты

1. Посетитель открывает главную страницу (`/`)
2. Видит секцию «Проекты» с карточками
3. Нажимает «Все проекты» → переходит на `/projects/`
4. Видит грид проектов с кнопками-фильтрами по категориям (Компрессорные станции, Магистральные газопроводы, Хранилища газа)
5. Нажимает категорию → проекты фильтруются
6. Кликает на карточку проекта → переходит на `/projects/<slug>`
7. Видит полное описание проекта, изображение, локацию, год

### 2. Посетитель читает блог

1. Посетитель переходит на `/blog/` (из навигации)
2. Видит грид из карточек статей (до 9 на странице)
3. Каждая карточка содержит: изображение, дату, заголовок, excerpt, время чтения
4. При наличии >9 статей внизу отображается пагинация
5. Нажимает на карточку → переходит на `/blog/<slug>`
6. Видит полную статью с hero-изображением, заголовком, датой, HTML-контентом
7. Хлебные крошки: Главная / Блог / Название статьи

### 3. Соискатель ищет работу

1. Соискатель переходит на `/vacancies/` (из навигации)
2. Видит список вакансий и теги-фильтры по локации (Калининград, ЯНАО, Москва)
3. Нажимает тег «ЯНАО» → показываются только вакансии в ЯНАО
4. Кликает на вакансию → переходит на `/vacancies/<id>`
5. Видит: название, локацию, тип занятости, зарплату, описание, требования
6. Внизу — форма отклика с полями: ФИО, телефон, email, сопроводительное письмо
7. Заполняет форму → данные отправляются через `POST /api/contact`

### 4. Посетитель отправляет заявку

1. Посетитель на главной странице прокручивает до секции «Контакты»
2. Видит контактную форму: имя, телефон (маска +7), email, сообщение
3. Заполняет поля → JS-валидация проверяет обязательные поля
4. Нажимает «Отправить» → AJAX-запрос `POST /api/contact`
5. При успехе — уведомление «Заявка успешно отправлена!»
6. При ошибке — сообщение с описанием проблемы

### 5. Админ управляет контентом

1. Админ переходит на `/admin/login`
2. Вводит логин и пароль → авторизация через Flask-Login
3. Попадает на Dashboard (`/admin/`) со статистикой
4. Выбирает раздел (Блог / Вакансии / Проекты / Заявки / Пользователи)
5. Создаёт/редактирует/удаляет записи через Flask-Admin
6. Для блога и вакансий — CKEditor (WYSIWYG-редактор)
7. Slug для статей и проектов генерируется автоматически при создании
8. Просматривает заявки (read-only), отмечает прочитанными

## Интеграции

На текущий момент проект **не имеет внешних интеграций**:

- Нет CRM (заявки хранятся в SQLite)
- Нет платёжных систем
- Нет аналитики (Google Analytics, Яндекс.Метрика)
- Нет email-рассылок
- Нет интеграции с соцсетями (авторизация)
- Нет интеграции с hh.ru или другими джоб-бордами

Все данные хранятся локально в базе SQLite.
