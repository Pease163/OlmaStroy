# Ограничения И Техдолг

Документ отражает реальные ограничения текущей кодовой базы, а не исторические гипотезы.

## Текущие Ограничения

### 1. Тестирование

- В репозитории нет unit, integration и e2e тестов.
- Базовая верификация сейчас строится на ручном проходе и smoke-командах.

### 2. Разграничение Доступа

- Модели `Role` и `Permission` уже существуют.
- UI ролей и permissions уже есть.
- Но большинство admin API маршрутов защищено только через `@admin_required`.
- Декоратор `permission_required()` пока не внедрён системно.

### 3. Два Административных Контура

- `/panel` является основной админкой.
- `/admin` всё ещё живёт рядом и использует другой стек авторизации.
- Это увеличивает стоимость поддержки и требует держать в голове два интерфейса.

### 4. Зависимости От CDN

Публичный контур и legacy admin всё ещё используют внешние CDN:

- Google Fonts;
- `https://cdn.tailwindcss.com`;
- CKEditor CDN для legacy Flask-Admin форм.

### 5. База Данных По Умолчанию

- По умолчанию используется SQLite.
- Для production-нагрузки это слабое место по конкурентной записи и масштабированию.

### 6. Локальное Хранилище Медиа

- Uploads хранятся в `app/static/uploads/`.
- Нет вынесенного object storage.
- Нет версионирования и дедупликации файлов.

### 7. Ограниченный Публичный API

- Во внешний JSON API вынесена только contact form.
- Всё остальное для публичного сайта рендерится server-side.

## Ближайший Техдолг

### Высокий Приоритет

- Ввести автоматические тесты для backend и `admin-panel`.
- Перевести production-сценарий с SQLite на PostgreSQL.
- Последовательно внедрить route-level RBAC через `permission_required()`.
- Определить план вывода из эксплуатации legacy `/admin`.

### Средний Приоритет

- Убрать CDN-зависимость Tailwind и CKEditor в legacy-контуре.
- Вынести uploads в persistent или внешнее хранилище.
- Добавить CI для smoke checks, TypeScript и lint.
- Формализовать релизную сборку SPA и серверный deploy pipeline.

### Низкий Приоритет

- Оптимизировать тяжёлые статические активы, включая `app/static/img/logo-icon.svg` (~296 KB).
- Добавить экспорт или документацию для внутренних operational workflows.
- Упростить поддержку двух UI-контуров после консолидации функциональности.

## Риски Для Production

| Риск | Почему это важно |
|------|------------------|
| Нет автотестов | Регрессии ловятся поздно и вручную |
| SQLite по умолчанию | Ограничивает конкурентную запись |
| Два admin-контура | Сложнее поддерживать безопасность и UX |
| RBAC не доведён до route-level | Роли в UI и роли на backend пока не полностью эквивалентны |
| Медиа на локальном диске | Риск потери файлов и сложность горизонтального масштабирования |

## Что Уже Исправлено По Сравнению Со Старыми Доками

- `SECRET_KEY` и `JWT_SECRET_KEY` больше не имеют статического fallback, а генерируются случайно при отсутствии переменных окружения.
- Таблица транслитерации вынесена в `app/utils/slug.py`, а не дублируется по моделям.
- Flask-Migrate и реальные миграции уже есть в репозитории.
- Основной admin UI это `/panel`, а не Flask-Admin.
