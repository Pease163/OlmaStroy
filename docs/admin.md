# Админка `/panel`

Этот документ описывает рабочие сценарии основной административной панели. Подробности внутренней реализации SPA см. в [admin-panel.md](admin-panel.md).

## Вход

- URL: `/panel/login`
- seed login: `admin`
- seed password: `admin123`

После успешного логина пользователь попадает в `/panel`.

Если у пользователя включена 2FA:

1. сначала вводятся логин и пароль;
2. backend отвечает `requires_2fa`;
3. пользователь вводит TOTP-код из приложения-аутентификатора.

## Общий Паттерн Работы

Большинство модулей устроены одинаково:

- список с поиском и пагинацией;
- кнопка создания записи;
- форма создания/редактирования;
- toast-уведомления об успехе или ошибке;
- удаление через confirm dialog.

Для сущностей с публикацией или видимостью есть быстрые статусы и bulk-операции на backend.

## Разделы Подробно

### Дашборд

Показывает:

- число постов;
- число вакансий;
- число проектов;
- число заявок;
- число пользователей;
- графики по публикациям, заявкам, типам вакансий и категориям проектов;
- последние действия из audit log.

### Блог

Позволяет:

- создавать статьи;
- редактировать HTML-контент;
- указывать excerpt и изображение;
- управлять публикацией;
- задавать SEO-поля;
- привязывать теги.

### Проекты

Позволяет:

- редактировать title/slug/location/category/year;
- управлять видимостью на публичном сайте;
- задавать порядок сортировки;
- наполнять описание и контент;
- вести галерею изображений проекта.

### Вакансии

Позволяет:

- управлять активностью вакансий;
- редактировать зарплату и требования;
- выбирать тип занятости.

### Медиа

Раздел поддерживает:

- загрузку одного или нескольких файлов;
- поиск по имени файла;
- просмотр grid/list;
- копирование URL;
- удаление файла.

Файлы физически лежат в `app/static/uploads/`.

### Заявки

Оператор может:

- искать заявки по имени, email, телефону и теме;
- открыть detail view;
- отметить заявку прочитанной;
- удалить заявку.

### Пользователи И Роли

Доступны:

- создание пользователя;
- назначение роли;
- активация/деактивация;
- смена пароля;
- просмотр и управление ролями;
- просмотр permissions, сгруппированных по областям.

Важно: фактическое разграничение доступа на backend сейчас в основном основано на `is_admin`, даже если роли и permissions уже настроены.

### Настройки

Редактор может менять `site_settings`, сгруппированные по категориям.

Типы значений:

- string
- text
- boolean
- number
- json

### Аудит

Позволяет просматривать:

- кто и когда логинился;
- какие сущности создавал, менял и удалял;
- IP-адрес и название объекта.

### Уведомления

Доступны:

- список уведомлений;
- mark as read;
- mark all read;
- удаление уведомления.

### Профиль И 2FA

На странице профиля пользователь видит:

- логин;
- email;
- назначенную роль;
- статус администратора;
- состояние двухфакторной аутентификации.

Пользователь может:

- включить 2FA через QR-код;
- подтвердить настройку 6-значным кодом;
- отключить 2FA, подтвердив действие паролем.

## Глобальный Поиск

Из header доступен глобальный поиск:

- открывается по `Ctrl+K` или `Cmd+K`;
- ищет по блогам, вакансиям, проектам и заявкам;
- ведёт сразу в нужный раздел SPA.

## Legacy `/admin`

Старый Flask-Admin по адресу `/admin/` остаётся в проекте, но не считается основным рабочим интерфейсом.

Когда он полезен:

- резервный доступ при проблемах SPA;
- старые ручные операции;
- быстрая проверка нескольких базовых сущностей.

Когда его не стоит считать источником истины:

- новые сущности из admin API v2;
- роли и permissions;
- медиа-библиотека;
- settings, audit, notifications, 2FA, search.
